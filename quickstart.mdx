# Quickstart


#### Learn how to create your first project, add tasks, and manage their status.

## Prerequisites

Before you begin, make sure you have:

- A valid API key (format: `tf-live-xxxxxxxxxxxxxxxx`)
- A tool to make HTTP requests — Postman or any HTTP client ⚠ **Keep your API key private.**

<Warning>
  Keep your API key private.
</Warning>

## Workflow overview

Task Flow follows a strict dependency order. You must create a project before creating tasks.

```text
POST /projects → POST /tasks → PATCH /tasks/{id} → DELETE /tasks/{id}
```

## Step 1: Create a project

Every task belongs to a project. Start by creating one. Save the `project_id` from the response — you'll need it in the next step.

```bash
# Replace {api_key} with your actual API key
curl -X POST https://test-writechoice.onrender.com/projects \
  -H "Authorization: Bearer {api_key}" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "API Documentation Revamp",
    "description": "Rewriting all public-facing API docs."
  }'
```

**Response** `201 Created`

```json
{
  "project_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "name": "API Documentation Revamp",
  "description": "Rewriting all public-facing API docs.",
  "created_at": "2025-05-12T10:30:00.000Z"
}
```

<Note>
  Copy the `project_id` value. You will need it to create tasks in the next step.
</Note>

## Step 2: Create a task

Use the `project_id` from step 1 to add a task to your project. Tasks are created with status `pending` by default.

```bash
curl -X POST https://test-writechoice.onrender.com/tasks \
  -H "Authorization: Bearer {api_key}" \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "title": "Write introduction page",
    "description": "Draft the overview and getting started sections."
  }'
```

**Response** `201 Created`

```json
{
  "task_id": "f9e8d7c6-b5a4-3210-fedc-ba9876543210",
  "project_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "title": "Write introduction page",
  "status": "pending",
  "created_at": "2025-05-12T10:35:00.000Z",
  "updated_at": "2025-05-12T10:35:00.000Z"
}
```

## Step 3: List tasks in a project

Retrieve all tasks belonging to a project using its `project_id` in the URL path.

```bash
curl https://test-writechoice.onrender.com/projects/a1b2c3d4-e5f6-7890-abcd-ef1234567890/tasks \
  -H "Authorization: Bearer {api_key}"
```

**Response** `200 OK`

```json
{
  "project_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "total": 1,
  "tasks": [ ... ]
}
```

## Step 4: Update a task

Use `PATCH /tasks/{task_id}` to update the status, title, or description of a task. At least one field is required.

```bash
curl -X PATCH https://test-writechoice.onrender.com/tasks/f9e8d7c6-b5a4-3210-fedc-ba9876543210 \
  -H "Authorization: Bearer {api_key}" \
  -H "Content-Type: application/json" \
  -d '{ "status": "in_progress" }'
```

<Note>
  Valid status values: `pending`, `in_progress`, `done`. Any other value returns a `400` error.
</Note>

## Step 5: Delete a task

Permanently remove a task with `DELETE /tasks/{task_id}`.

```bash
curl -X DELETE https://test-writechoice.onrender.com/tasks/f9e8d7c6-b5a4-3210-fedc-ba9876543210 \
  -H "Authorization: Bearer {api_key}"
```

<Warning>
  Deletion is permanent. There is no way to recover a deleted task.
</Warning>
